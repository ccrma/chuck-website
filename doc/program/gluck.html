Extensions : GlucK

version: 1.1.x (frankenstein)

GlucK is Chuck's interface to OpenGL and glut. We provide a general 'wrapper' for these libraries, while adding a few convenience calls for the sake of code brevity ( so far, not enough ). We've provided only a subset of GL/GLUT for the moment - once we have array and object support, we will include the rest of the functions, as well as provide convenience objects within GlucK for drawing and interaction

[importing the gl/glu/gluck namespaces]
"gl.ckx" => (:gl:); "glu.ckx" => (:glu:); "gluck.ckx" => (:gluck:);

The general form of the calls is quite similar to the standard functions, when using the import code given above, is shown in the example below.

If not otherwise noted, a function's behavior will be identical to that described in standard OpenGL documentation.

Note: Since chuck's 'float' type corresponds to 'double', we use the 'f' notation in GlucK, but the call uses the 'd' option, if available.

[GlucK => GL translation]
standard | gl.ckx glFunctionName(args...); gl.FunctionName(args...); glFunction(GL_CONSTANT); gl.Function(gl.CONSTANT); glBegin(GL_LINES) gl.Begin(gl.LINES); glVertex2f ( 0.0, 0.0 ) gl.Vertex2d(0.0, 0.0); glVertex2f ( 1.0, 1.0 ) gl.Vertex2d(1.0, 1.0); glEnd(); gl.End();

namespace: Machine

ChucK runtime interface to the virtual machine. this interface can be used to manage shreds. They are similar to the On-the-fly Programming Commands, except these are invoked from within a ChucK program, and are accessible to the timing mechanism.

----------

[function]: int add ( string path );

  • compile and spork a new shred from file at 'path' into the VM now
  • returns the shred ID
  • (see example/machine.ck)

[function]: int spork ( string path );

  • same as add

[function]: int remove ( int id );

  • remove shred from VM by shred ID (returned by add/spork)

[function]: int replace ( int id, string path );

  • replace shred with new shred from file
  • returns shred ID , or 0 on error

[function]: int status ( );

  • display current status of VM
  • (see example/status.ck)
